VitalOrgans Emergency Response

React | Node.js | Express.js | PostgreSQL | Tailwind CSS | AWS S3 | Git | Jira

VitalOrgans aims to enhance emergency response by providing hospitals access to patient's medical history via patient's registered email. It works as a tool for patient to manage their medical reports, medical details and their personal information.

GitHub

Live

Video

Emergency Response: Implemented a secure authentication system for licensed hospitals with a LICENSE KEY (HSP-####-####-IN) using JWT-based middleware to verify tokens and enforce role-based access control. This ensures secure, compliant access to patient data via email, eliminating paperwork for quick, accurate care.

Data Security: Utilized AWS S3 to securely upload, store, and manage PDF reports. By leveraging S3's scalable storage and CORS policies, ensured that even large PDF files are securely handled and easily accessible.

Health Awareness: Utilized APEXCHARTS to create dynamic bar graphs for visualizing vital stats, applying conditional colors, using fill colors property to indicate normal (green) and out-of-limit (red) values. This helps users quickly assess their health metrics.

Emergency Contact Integration: Integrated emergency contact features to quickly notify loved ones, providing peace of mind during critical times.

PharmaKeeper MedicineKit

JavaScript | HTML | CSS | Git | Web Speech API

Personal Medicine Manager that tracks every medicine, its brand name, expiry date, quantity, and description with ease. It keeps users' health in check by staying organized and informed about their medications at all times.

GitHub

Live

Medicine Management: Users can easily add new medicines, edit existing entries, and delete medicines. Users can upload images of their medicines to help with identification.

Expiry Date Tracking: App uses the Day.js library to handle date manipulation and formatting. It displays medicines with the earliest expiry dates and highlights expired or soon-to-expire items with the uses of color with red(expired), green(expiring today), gray(not expired).

Speech Recognition: Integrates the Web Speech API to allow users to input data using voice commands, enhancing accessibility and ease of use.

Local Storage: All medicine data is stored in the browser's local storage, ensuring that user information is retained between sessions without the need for a backend server.
